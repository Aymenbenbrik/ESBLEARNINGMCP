## Problèmes identifiés et corrections

### 1. `SyllabusTNService.extract_text()` — tables DOCX ignorées
**Fichier :** `ESB-main/app/services/syllabus_tn_service.py`

**Cause :** L'extracteur DOCX ne lisait que `doc.paragraphs` (3 paragraphes, ~400 chars). Tout le contenu réel d'une fiche module TN est dans un tableau Word fusionné — AA, chapitres, évaluation, bibliographie — était complètement ignoré.

**Correction :** Réécriture de la branche DOCX dans `extract_text()` :
- Parcours de tous les tableaux du document
- Déduplication par pointeur XML (`cell._tc`) pour gérer les cellules fusionnées
- Résultat : **6 187 chars** extraits (vs 400 avant), contenant les AA, chapitres, évaluation

```python
# Avant
if ext in ["doc", "docx"]:
    d = docx.Document(path)
    return "\n".join([clean_utf(p.text) for p in d.paragraphs])

# Après
if ext in ["doc", "docx"]:
    d = docx.Document(path)
    parts = []
    para_text = "\n".join([clean_utf(p.text) for p in d.paragraphs if p.text.strip()])
    if para_text:
        parts.append(para_text)
    for tbl in d.tables:
        seen_ptrs = set()
        for row in tbl.rows:
            row_parts = []
            for cell in row.cells:
                ptr = cell._tc
                if ptr in seen_ptrs:
                    continue
                seen_ptrs.add(ptr)
                ct = clean_utf(cell.text.strip())
                if ct:
                    row_parts.append(ct)
            if row_parts:
                parts.append(" | ".join(row_parts))
    return "\n".join(parts)
```

### 2. GET `/api/v1/syllabus/:id` — données TN normalisées non sérialisées
**Fichier :** `ESB-main/app/api/v1/syllabus.py`

**Cause :** L'endpoint GET retournait uniquement `tn_data` (blob JSON brut), sans sérialiser les tables ORM normalisées (`tn_aa`, `tn_chapters`, `tn_evaluation`, `tn_bibliography`, `tn_admin`).

**Correction :**
- Ajout de la fonction `_serialize_tn_syllabus(syllabus)` qui sérialise toutes les relations ORM TN en un objet structuré
- Ajout du champ `tn_structured` dans la réponse JSON du GET endpoint

Structure retournée :
```json
{
  "tn_structured": {
    "administrative": { "module_name", "code_ue", "department", "credits", ... },
    "aa": [{ "number": 1, "description": "..." }],
    "aap": [{ "number": 3, "selected": true }],
    "chapters": [
      {
        "index": 1,
        "title": "Chapitre I : Calculs algébriques",
        "aa_links": [],
        "sections": [{ "index": "1.1", "title": "...", "aa_links": [] }]
      }
    ],
    "evaluation": { "final_grade_formula": "30% CC + 70% Examen Final", ... },
    "bibliography": [{ "position": 1, "entry": "..." }]
  }
}
```

### 3. `SyllabusViewer.tsx` — aucun affichage pour les syllabus TN
**Fichier :** `esb-nextjs/components/courses/SyllabusViewer.tsx`

**Cause :** Le composant ne gérait que le format BGA (`clo_data`, `plo_data`, `weekly_plan`). Pour un syllabus TN, il affichait toujours "pas encore extrait".

**Correction :** Ajout du composant `TNSyllabusView` avec :
- **Header administratif** : code UE, département, filière, option, volume, crédits, responsable, enseignants
- **Onglet "Acquis d'Apprentissage"** : tableau AA1→AAn avec descriptions
- **Onglet "Contenu"** : chapitres pliables avec sections et liens AA
- **Onglet "Évaluation"** : formule de notation, méthodes, critères, mesures
- **Onglet "Bibliographie"** : références numérotées
